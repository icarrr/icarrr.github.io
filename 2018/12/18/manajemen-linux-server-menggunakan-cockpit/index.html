Manajemen Linux Server Menggunakan Cockpit

Cockpit?

Apa sih cockpit itu? Bukannya cockpit itu ruang khususnya pilot?

Cockpit adalah sebuah aplikasi adminstrasi server berbasi web. Dengan cockpit, grafik penggunaan cpu, memory, network, hdd dapat dilihat langsung pada saat autentikasi user cockpit berhasil serta dapat mengontrol service yang dijalankan melalui SystemD.

Cockpit berjalan di port 9090, tidak membutuhkan web server jadi jika anda belum menginstall web server apapun, itu tidak menjadi masalah, serta resource yang digunakan cockpit sangat sedikit karena service cockpit akan dijalankan secara otomatis jika anda mencoba mengaksesnya melalui web page dan setelah beberapa menit setelah anda logout secara otomatis service cockpit akan di matikan.

Bermain Cockpit

Untuk pengguna RHEL/CentOS 7

1) Buka shell kemudian login sebagai superuser (root).

2) Install cockpit dengan command yum install cockpit.

3) Aktifkan socket cockpit dengan command systemctl enable --now cockpit.socket.

4) Berikan akses dari firewall dengan command berikut:

root@sibunglonlabs# firewall-cmd --add-service=cockpit
root@sibunglonlabs# firewall-cmd --add-service=cockpit --permanent

5) Sekarang cockpit sudah bisa diakses melalui web page melalui url sibunglon.com:9090/ atau IP_server:9090/

Untuk pengguna Ubuntu 16 & 18

1) Buka shell kemudian login sebagai superuser (root).

2) Install cockpit dengan command apt-get install cockpit.

3) Sekarang cockpit sudah bisa diakses melalui web page, contohnya melalui url sibunglon.com:9090/ atau IP_server:9090/
